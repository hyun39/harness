## AI Agent 스택 (ADR-0018)

```
┌─────────────────────────────────────────────────────────┐
│ apps/agent/  (FastAPI + LangServe)                      │
│                                                         │
│   ├─ LangServe        — /invoke, /batch, /stream, /play │
│   ├─ LangGraph        — StateGraph (다단계 오케스트레이션)│
│   └─ LangChain Core   — Prompt, LLM, Parser, Tool       │
│                          ↓                              │
│   Gemini / Embedding (외부 LLM API — agent 내부에서만)    │
└─────────────────────────────────────────────────────────┘
                ▲
                │  httpx · LangServe Client SDK
                │
┌──────────────┴──────────────┐
│ apps/api · apps/airflow ... │  ← LLM SDK 직접 import 금지
└─────────────────────────────┘
```

**호출 단일 채널 강제**:
- 다른 서비스(api, airflow, frontend)는 LLM SDK 의존성을 갖지 않는다
- agent의 LangServe endpoint를 통해서만 LLM 기능을 사용한다
- CI/린트로 비-agent 컴포넌트의 LLM SDK import를 차단한다

## LangGraph StateGraph 패턴 (오케스트레이션)

> ADR-0017에 따라 분기·재시도·중간 상태가 필요한 흐름은 **LangGraph StateGraph**로 표현한다.
> 단일 LLM 호출로 끝나는 단순 흐름은 LangChain Core 체인(아래)도 허용.

```python
# apps/agent/app/graph/agent_graph.py 패턴
from typing import TypedDict
from langgraph.graph import START, END, StateGraph

class AgentState(TypedDict):
    query: str
    trade_date: str
    retrieved_events: list[dict]
    answer: str | None

def retrieve(state: AgentState) -> dict:
    events = vector_store.search(state["query"], k=5, score_threshold=0.7)
    return {"retrieved_events": events}

def generate(state: AgentState) -> dict:
    response = llm.invoke(prompt.format(
        query=state["query"],
        context=state["retrieved_events"],
    ))
    return {"answer": response.content}

graph = StateGraph(AgentState)
graph.add_node("retrieve", retrieve)
graph.add_node("generate", generate)
graph.add_edge(START, "retrieve")
graph.add_edge("retrieve", "generate")
graph.add_edge("generate", END)

agent = graph.compile()
result = agent.invoke({"query": "최근 FOMC 결과", "trade_date": "2026-05-05"})
```

**핵심 규칙**:
- 모든 상태는 `AgentState` TypedDict에 명시 — 자유로운 필드 추가 금지
- 노드는 순수 함수에 가깝게(외부 의존은 클로저·DI로 주입) 작성 → 단위 테스트 가능
- 분기는 `add_conditional_edges`로 표현 (if/elif 노드 내 분기 자제)
- 무한 루프 방지: `graph.compile(recursion_limit=N)` 명시

## LangChain Core 체인 패턴 (단일 호출)

```python
# 단순 1-step 변환: PromptTemplate | LLM | PydanticOutputParser
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import PydanticOutputParser
from langchain_google_genai import ChatGoogleGenerativeAI  # ADR-0011 Gemini

parser = PydanticOutputParser(pydantic_object=Summary)

chain = (
    PromptTemplate(
        template="...\n{format_instructions}",
        partial_variables={"format_instructions": parser.get_format_instructions()},
    )
    | ChatGoogleGenerativeAI(model="gemini-2.0-flash", temperature=0)
    | parser
)

result: Summary = await chain.ainvoke({"information": data})
```

# Cache-Aside: 벡터 스토어 캐시 조회 → Miss 시 LLM 실행
async def get_analysis(name: str):
    cached = await vectorstore.similarity_search(name, k=1, score_threshold=0.9)
    if cached:
        return cached[0].metadata["result"]

    result = await run_pipeline(name)
    vectorstore.add_documents([Document(
        page_content=name,
        metadata={"result": result, "created_at": now()}
    )])
    return result

## LLM 비용 관리 원칙

| 원칙 | 구현 |
|------|------|
| 결과 캐싱 | Redis 7일 TTL |
| 토큰 사용량 추적 | OTel `llm.output_tokens` span attribute |
| temperature=0 | 결정론적 결과 (비용 예측 가능) |
| 재시도 제한 | `max_iterations=5` |
