# STD-12 — ADR 신규 작성 표준 (재프롬프트)

> **목적**: 새 기술 결정 시 `docs/adr/ADR-NNNN-*.md` 를 GOV-02 형식에 맞게  
> 정확하게 작성하는 표준 프롬프트를 정의한다.  
> 작성 의무 기준: [GOV-02 ADR 정책](../gov/02_adr.md)

---

## 1. ADR 작성 대상 판단 기준

| 상황 | ADR 필요 | 예시 |
|------|---------|------|
| 새 라이브러리·프레임워크 도입 | ✅ 필수 | LangChain4j 채택, Redis 도입 |
| 기존 기술 교체 | ✅ 필수 | python-jose → PyJWT (ADR-0012) |
| 아키텍처 패턴 변경 | ✅ 필수 | 모노리스 → MSA |
| 보안·데이터 정책 결정 | ✅ 필수 | PII 암호화, 토큰 보존 정책 |
| 버그 픽스 | ❌ 불필요 | — |
| 기존 패턴 내 기능 추가 | ❌ 불필요 | 새 API 엔드포인트 |
| 의존성 마이너 버전 업 | ❌ 불필요 | FastAPI 0.136 → 0.137 |

---

## 2. 파일 네이밍 규칙

```
docs/adr/ADR-{NNNN}-{kebab-case-title}.md

NNNN: 4자리 0-패딩. 다음 번호 확인:
  ls docs/adr/ | grep -v README | sort | tail -1

예:
  ADR-0019-redis-for-session-cache.md
  ADR-0020-celery-for-async-tasks.md
```

## 3. 재작성 프롬프트

결정 컨텍스트를 `[[ ]]` 에 채워서 Claude에게 전달한다.

````
다음 기술 결정에 대해 ADR을 작성해줘.
GOV-02 필수 섹션을 모두 포함하고, 저장 위치도 알려줘.

**결정 컨텍스트:**
- 결정 제목: [[ 짧고 명확한 제목 ]]
- 배경 (왜 결정이 필요했는가): [[ 상황·문제 설명 ]]
- 검토한 옵션 (최소 2개, 각 장단점 포함):
  옵션 1: [[ 옵션명 + 장점 + 단점 ]]
  옵션 2: [[ 옵션명 + 장점 + 단점 ]]
  옵션 3 (있으면): [[ ]]
- 최종 선택: [[ 선택한 옵션 ]]
- 선택 이유 (한 문장): [[ "우리는 X를 선택한다. 왜냐하면 Y이기 때문이다." ]]
- 관련 ADR (있으면): [[ ADR-NNNN ]]
- 관련 GOV/STD (있으면): [[ GOV-08, STD-07 ]]

**현재 ADR 번호 확인:**
ls docs/adr/ | grep -v README | sort | tail -1
(위 결과에서 다음 번호를 계산해서 파일명에 반영)

**출력 형식 (GOV-02 준수):**
````

## 4. ADR 작성 후 필수 후속 작업

```bash
# 1. ADR README 인덱스 업데이트
cat docs/adr/README.md  # 현재 목록 확인 후 추가

# 2. ARCHITECTURE.md 영향 섹션 갱신 (STD-10 프롬프트 참조)

# 3. 커밋
git add docs/adr/ADR-{NNNN}-*.md docs/adr/README.md
git commit -m "docs(adr): ADR-{NNNN} {결정 제목}"
```

## 5. 현행화 — 기존 ADR 수정이 필요한 경우

> **수락된 ADR은 내용을 소급 수정하지 않는다** (GOV-02 금지 사항).

| 상황 | 처리 방법 |
|------|---------|
| 결정이 번복됨 | 새 ADR 작성 후 기존 ADR 상태를 `대체됨(ADR-NNNN)` 으로 변경 |
| 구현 내용이 달라짐 | 기존 ADR 상태 유지 + 새 ADR에서 차이 설명 |
| 오탈자·날짜 오류 | 수정 허용 (내용 변경 아님) |

**기존 ADR 현행화 프롬프트** (오탈자·날짜·상태 수정 한정):

````
docs/adr/ADR-{NNNN}-*.md 파일을 읽고 아래 내용만 수정해줘.
(결정 내용 자체는 수정하지 않는다 — GOV-02)

수정 항목:
- [[ 수정할 항목: 예) "상태: 수락됨 → 대체됨(ADR-0020)" ]]
- [[ 수정할 항목 2 ]]

파일: docs/adr/ADR-{NNNN}-{title}.md
````

## 6. 검증 체크리스트

```
□ 파일명이 ADR-{NNNN}-{kebab-case}.md 형식인가
□ 번호가 기존 ADR 다음 번호인가 (중복·건너뜀 없음)
□ 필수 섹션 7개 모두 포함: 상태/날짜/결정자/배경/검토한옵션/결정/결과
□ 검토한 옵션이 최소 2개인가
□ 결과 섹션에 부정적 내용과 트레이드오프가 있는가
□ docs/adr/README.md 인덱스에 추가됐는가
□ ARCHITECTURE.md 관련 섹션 갱신됐는가 (STD-10 §4 트리거 확인)
```
